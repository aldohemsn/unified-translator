# 翻译方法论白皮书
## Translation Methodology Whitepaper

本文档总结并固化 Unified Translator 框架的核心翻译理论，源自三个独立项目的生产实践。

---

## 一、CIL 三维知识体系 (Legal Strategy)

### 理论基础
法律翻译的挑战在于：高精度要求、术语强约束、逻辑链完整性。

**CIL = Context + Insight + Logic**

```
┌─────────────────────────────────────────────────────────┐
│                    CIL 三维模型                          │
├─────────────────────────────────────────────────────────┤
│  Context (语境)                                         │
│  ├── 文档类型识别                                        │
│  ├── 受众分析                                           │
│  └── 语气基调                                           │
├─────────────────────────────────────────────────────────┤
│  Insight (洞见)                                         │
│  ├── 微领域定位 (如"香港知识产权诉讼")                    │
│  ├── 关键术语定义                                        │
│  └── 陷阱警示 (False Friends)                            │
├─────────────────────────────────────────────────────────┤
│  Logic (逻辑)                                           │
│  ├── 费曼技术：用反向语言解释                             │
│  ├── 逻辑链提取                                          │
│  └── 非逐词翻译，意译优先                                 │
└─────────────────────────────────────────────────────────┘
```

### 实施要点
1. **预处理阶段**生成 CIL 三组件，注入每个翻译请求。
2. **术语表强制**：使用 `strict` 模式，违反即标注。
3. **逐行处理**：法律文本每段独立，语义分段替代固定批次。

### 适用场景
✅ 合同、判决书、法规
✅ 高术语密度文本
✅ 需要逻辑一致性的长文档

## 二、译审分离双角色模式 (Academic Strategy)

### 理论基础
学术翻译的挑战在于：准确性与流畅性的平衡、跨行句子处理、出版级质量要求。

**核心理念：翻译与审校分离，互不干扰**

```
┌───────────────────────────────────────────────────────────┐
│                    双角色流水线                            │
│                                                           │
│   Source ───▶ [Literal Translator] ───▶ Draft            │
│                                           │               │
│                                           ▼               │
│              [Academic Editor] ◀──────────┘               │
│                     │                                     │
│                     ▼                                     │
│                  Final Output                             │
└───────────────────────────────────────────────────────────┘
```

### 角色定义

**角色 1: Literal Translator (直译者)**
- 职责：语义精准传递
- 约束：**压制润色冲动**，忠于原文结构
- 输出：可能略显生硬的准确译文

**角色 2: Academic Editor (学术编辑)**
- 职责：语言润色、学术规范
- 约束：**源语言盲审**，不参考原文结构
- 输出：符合目标语言学术习惯的终稿

### 跨行合并协议
```
原文行 1: "This is a sentence that"
原文行 2: "spans multiple rows."

处理结果:
行 1 Target: "这是一个跨越多行的句子。"
行 2 Target: "[已向上合并]"
```

### QA 三重检查
| 类型 | 定义 | 处理 |
|------|------|------|
| Omission | 源信息**完全**缺失 | 标注 `[[QA FLAG: Omission]]` |
| Misinterpretation | 事实性**直接矛盾** | 标注 `[[QA FLAG: Misinterpretation]]` |
| Hallucination | 凭空添加**实质性**信息 | 标注 `[[QA FLAG: Hallucination]]` |

> ⚠️ 风格变化、作者名音译差异、标点微调**不属于**错误。

### 适用场景
✅ 学术论文、专著
✅ 需要润色的人工初译
✅ 出版级质量要求

## 三、全语境约束模式 (Video Strategy)

### 理论基础
视频翻译的挑战在于：对话跳跃、语境碎片化、转录错误、翻译腔。

**核心理念：强语境约束 + 负向约束**

```
┌───────────────────────────────────────────────────────────┐
│                    视频翻译流水线                          │
│                                                           │
│   Full Transcript ───▶ [Compressor] ───▶ Scene Summary   │
│                                              │            │
│                                              ▼            │
│   [Style Guide Generator] ◀──────────────────┘            │
│              │                                            │
│              ▼                                            │
│   [Translator] + Blacklist + History ───▶ Output         │
└───────────────────────────────────────────────────────────┘
```

### 关键组件

**1. 场景摘要 (Compressed Context)**
```
主题：某科技产品发布会
说话人：主持人、CEO、技术演示者
语境：正式发布会，偶有轻松互动
关键术语：[产品名]、[技术名词]
```

**2. 风格指南 (Style Guide)**
- VO (Voice-Over): 流畅自然，可调整语序
- OS (On-Screen): 简洁精炼，保持对齐

**3. 转录审核 (Transcription Audit)**
- 检测 ASR 错误、同音错字、专名错误
- 标注：`⚠️ [TRANSCRIPTION FLAG]: 疑似"pay an arm"误转为"pay an RMB"`

**4. 翻译腔黑名单 (Translationese Blacklist)**
```yaml
blacklist:
  - "进行"  # 避免"进行讨论"，改用"讨论"
  - "通过"  # 避免"通过...方式"
  - "旨在"  # 书面语过重
  - "它"    # 中文常省略代词
```

### 适用场景
✅ 字幕翻译
✅ 视频脚本
✅ 采访/对话记录
✅ 需要口语化表达的内容

## 方法论对比

| 维度 | CIL (Legal) | 双角色 (Academic) | 全语境 (Video) |
|------|-------------|------------------|----------------|
| 核心理念 | 三维知识注入 | 译审分离 | 强语境约束 |
| 预处理 | 生成 C+I+L | 生成双 Persona | 生成摘要+指南 |
| 批次策略 | 语义分段 | 固定批次 | 固定批次 |
| 术语控制 | 严格强制 | 建议性 | 建议性+黑名单 |
| 后处理 | 术语校验 | QA 三重检查 | 转录审核 |

---

## 设计原则

1. **质量不可妥协**: 效率优化不得损害翻译质量。
2. **语境为王**: 无论何种策略，充分的上下文是准确翻译的基础。
3. **人机协作**: LLM 是辅助工具，最终决策权归译者。
4. **可追溯性**: 所有修改和标注应可追溯，便于人工复核。
